Typedef Documentation

◆ CHEAX

typedef struct cheax CHEAX

The type of the cheax virtual machine state, a pointer to wich is needed for most cheax API functions.

The virtual machine is initialized with cheax_init(), and destroyed with cheax_destroy().

See also
cheax_init(), cheax_destroy(), cheax_load_features(), cheax_load_prelude()

◆ cheax_load_feature()

int cheax_load_feature ( CHEAX *  c,
const char *  feat 
)

Loads extra functions or language features into the cheax environment, including 'unsafe' ones.

Supported values for feat are:

  • "file-io" to load fopen and fclose built-ins;
  • "set-max-stack-depth" to load the set-max-stack-depth built-in;
  • "gc" to load the gc built-in function;
  • "exit" to load the exit function;
  • "stdin" to expose the stdin variable;
  • "stdout" to expose the stdout variable;
  • "stderr" to expose the stderr variable;
  • "stdio" to expose stdin, stdout and stderr;
  • "all" to load every feature available (think twice before using).

A feature can only be loaded once. Attempting to load a feature more than once will cause no action.

Parameters
cVirtual machine instance.
featWhich feature to load.
Returns
0 if the given feature was loaded succesfully, or -1 if the given feature is not supported.

Sets the value of a symbol.

Sets cheax_errno() to

  • CHEAX_EAPI if id is NULL;
  • CHEAX_ENOSYM if no symol with name id could be found;
  • CHEAX_EREADONLY if the given symbol was declared read-only;
  • or CHEAX_ETYPE if symbol is synchronised and type of value is invalid.

cheax_set() cannot be used to declare a new symbol, use cheax_def() instead.
